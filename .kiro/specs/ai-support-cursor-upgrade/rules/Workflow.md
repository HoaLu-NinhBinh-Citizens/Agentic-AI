# AI_SUPPORT — Development Workflow Rules

====================================================================
CORE DEVELOPMENT PHILOSOPHY
===========================

Cursor is used ONLY as a development accelerator
for building the AI_SUPPORT local embedded intelligence system.

Cursor is NOT:

* the final engineering intelligence
* the runtime embedded AI system
* the authority for CAR firmware architecture
* the primary embedded engineering agent

Cursor exists ONLY to:

* accelerate implementation
* accelerate infrastructure development
* accelerate architecture construction
* accelerate tooling creation
* help build AI_SUPPORT

====================================================================
PRIMARY PROJECT OBJECTIVE
=========================

The real objective of this project is:

Build a local AI agent capable of:

* understanding embedded systems
* understanding MCU architecture
* understanding firmware repositories
* understanding hardware semantics
* reasoning deterministically
* validating embedded correctness
* assisting automotive firmware development

The final intelligence layer must belong to:

* AI_SUPPORT local agent

NOT:

* external AI coding assistants

====================================================================
FINAL ENGINEERING WORKFLOW
==========================

Target Workflow:

Engineer
→ AI_SUPPORT Local Agent
→ Semantic Reasoning
→ Validation
→ Firmware Modification
→ Embedded System Integration

NOT:

Engineer
→ Cursor directly edits CAR firmware

====================================================================
CODE OWNERSHIP RULES
====================

ONLY AI_SUPPORT is allowed to:

* modify CAR firmware
* modify automotive architecture
* modify hardware configuration
* generate embedded firmware updates
* perform hardware validation
* reason about embedded execution
* manage deterministic engineering workflows

Cursor must NEVER become:

* direct firmware authority
* primary embedded reasoning engine
* production embedded intelligence layer

====================================================================
AI_SUPPORT RESPONSIBILITIES
===========================

AI_SUPPORT must eventually handle:

# Code Intelligence

* repository understanding
* symbol analysis
* call graph analysis
* architecture reasoning
* dependency tracking

# Hardware Intelligence

* RM understanding
* register semantics
* peripheral dependencies
* GPIO/DMA/IRQ reasoning
* clock tree analysis

# Embedded Validation

* initialization validation
* timing validation
* interrupt validation
* DMA validation
* hardware constraint validation

# Automotive Engineering

* CAN/LIN analysis
* OTA reasoning
* diagnostics workflows
* AUTOSAR-oriented architecture support

====================================================================
ENGINEERING PRINCIPLES
======================

1. ALWAYS prioritize deterministic correctness
2. ALWAYS validate hardware dependencies
3. ALWAYS reason structurally
4. ALWAYS maintain hardware traceability
5. ALWAYS prefer semantic understanding over token prediction
6. ALWAYS preserve embedded system reliability
7. ALWAYS explain hardware assumptions
8. NEVER hallucinate hardware behavior
9. NEVER bypass validation
10. NEVER allow uncontrolled firmware generation

====================================================================
KNOWLEDGE-FIRST ARCHITECTURE
============================

AI_SUPPORT must prioritize:

Knowledge Architecture

>

Prompt Engineering

The system should rely on:

* semantic graphs
* structured knowledge
* deterministic validation
* dependency reasoning
* hardware models
* execution flow understanding

NOT:

* shallow prompting
* random code generation
* autocomplete-only behavior

====================================================================
LONG-TERM SYSTEM TARGET
=======================

AI_SUPPORT should evolve into:

Reference Manual
→ Semantic Extraction
→ Hardware Knowledge Graph
→ Deterministic Validation
→ Embedded Reasoning Engine
→ Automotive Intelligence Platform

The final mission is:

Transform embedded AI from:

* probabilistic code generation

Into:

* deterministic embedded engineering intelligence.

====================================================================
FINAL RULE
==========

Cursor is used to BUILD the intelligence.

AI_SUPPORT becomes the intelligence.
